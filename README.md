# SparkleStudy 

一款基於 **遊戲化** 與 **非同步社交** 的沉浸式溫習輔助 App。

針對 DSE 及大學生在長時間、獨自備考環境下所面臨的孤獨感、注意力分散及缺乏即時回饋等問題，本項目旨在透過 2D 療癒系美術視覺、行為養成機制與非同步社交，打造一個低壓力且具高黏著度的學習陪伴平台。

## 🌟 核心設計理念

- **視覺減壓**：採用 2D 療癒系美術方向，有效降低學習焦慮。
- **資產量化**：將抽象的專注時間轉化為可視化的虛擬資產，如代幣與養分。
- **低壓社交**：透過「漂流瓶」與「虛擬自修室」提供群體陪伴感，同時避免過度的社交干擾。
- **心理激勵**：利用「即時獎勵」（裝扮解鎖）與「損失規避」（花卉休眠）機制提升學習效率。

## ✨ 主要功能

### ⏳ 專注與量化管理
- **番茄鐘系統**：支持自定義專注與休息時長，並提供白噪音與環境音效。
- **防作弊交互**：專注結束後需進行隨機位置的互動校驗，確保用戶真實投入而非掛機或睡眠。
- **數據儀表板**：自動統計每日及每週學習時數、目標達成率，並以圖表形式量化成長，提供改善建議。

### 🌱 虛擬角色與種植系統
- **2D 虛擬角色**：用戶可使用溫習獲得的代幣購買服飾與配件，打造個性化形象，且系統不設任何「課金」機制。
- **花卉種植**：專注期間產出的「養分」可用於培養花卉，經歷種子到盛開的全過程。
    - **策略選擇**：提供 3、7、21 日等不同週期的花卉，適應不同備考節奏。
    - **休眠機制**：若兩天未登錄，花卉將進入「休眠」狀態。過渡為非不可逆的死亡，以減少用戶心理壓力。
- **成就與收集**：設有展示牆與圖鑑，紀錄解鎖過的品種與稀有服飾。同時包含成就系統，如累計專注時數或連續登入獎勵。

### ✉️ 社交互助與激勵
- **匿名漂流瓶**：完成專注階段後可投放/接收匿名隨機鼓勵文字漂流瓶。
    - 後端整合 AI 情感分析與關鍵字過濾，防止釣魚、廣告、連結。
    - 亦提供一些標準句子，供用戶按下直接分享。
- **虛擬自修室**：顯示其他正在專注的用戶虛擬角色。
    - 僅限簡單的點讚動作，強調「共同奮鬥」的氛圍而非社交負擔。
    - 亦可以開放如 DSE 專區、Final Exam Period 之類的自修室，增添共同奮鬥氛圍
- 每天完成溫習後，可以按照時間獲得代幣、養分獎勵。

### 每日任務
- 連續登入獎勵： 鼓勵養成開啟習慣、有新衣服/代幣獎勵。
- 多元化任務：涵蓋專注時數達標、灌溉花卉、助人（發送漂流瓶/鼓勵他人）等多維度
激勵。

## 🛠 技術架構

本項目採用 **Compose Multiplatform (CMP)** 進行跨平台開發：

- **UI/UX 技術**：使用 CMP 的聲明式 UI 處理多層級 2D 貼圖堆疊與骨骼動畫。
- **後端服務**：
    - **Firebase Auth**：處理 Google 及 Apple 第三方登入。
    - **Express.js**：處理用戶鑒權、虛擬購買及數據更新。
    - **PostgresDB**：存儲用戶核心數據。
- **本地數據**：使用 **SQLDelight** 進行跨平台本地數據庫管理。
- **安全機制**：於後端部署輕量級 AI 模型（如 **Gemma** 或 **Qwen**）進行漂流瓶內容審查。

## 🚧 挑戰與解決方案

### 1. 離線與同步機制
溫習紀錄會先存儲於本地資料庫，待重新上線後自動與後端伺服器同步。若偵測到多設備登入衝突，將要求重新驗證以確保數據一致。

### 2. 後台偵測與專注維護
App 進入後台時會暫停計時。系統提供 30 秒寬限期供用戶回覆緊急訊息，若超時則會倒扣獎勵代幣以維持專注強度。

### 3. 素材處理
鑑於團隊規模限制，將利用開源素材庫/嘗試使用圖像生成式模型（如 **Stable Diffusion**）協助開發 2D 美術資源。
